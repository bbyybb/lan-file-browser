## 交互式启动引导

直接运行 `python file_browser.py`（不带任何参数）时，程序会进入交互式引导，逐步配置启动选项。每个问题直接按回车即使用括号内的默认值。

```
==================================================
  LAN File Browser - 启动配置
  直接按回车使用 [默认值]
==================================================

  端口 [25600]:                        ← 回车用 25600，或输入其他端口号

  密码模式:
    1. 自动生成随机密码（默认）
    2. 自定义固定密码
    3. 不设密码
    4. 多用户多密码
  请选择 [1]:                          ← 输入数字选择，回车用默认

  访问范围:
    1. 不限制，可访问所有文件（默认）
    2. 只允许访问指定目录
  请选择 [1]:                          ← 选 2 后输入目录路径（逗号分隔多个）

  阻止系统睡眠（服务运行期间防止电脑休眠）:
    1. 是，阻止睡眠（默认）
    2. 否，允许正常睡眠
  请选择 [1]:
```

**跳过引导的三种方式：**

| 方式 | 命令 | 说明 |
|------|------|------|
| 全用默认值 | `python file_browser.py -y` | 跳过所有问题直接启动 |
| 指定参数 | `python file_browser.py --roots D:/shared` | 传了参数自动跳过引导 |
| 组合参数 | `python file_browser.py --port 8080 --password abc -y` | 指定参数 + 强制跳过 |

## 目录白名单（沙箱模式）

默认情况下，程序允许访问电脑上的所有文件。如需限制可访问范围，有两种方式：

**方式一：命令行参数（推荐，临时生效）**

```bash
python file_browser.py --roots D:/shared-files E:/projects

# macOS 示例
python3 file_browser.py --roots /Users/bob/Public /tmp/shared
```

**方式二：修改配置文件（永久生效）**

修改 `file_browser.py` 顶部：

```python
ALLOWED_ROOTS = ["D:/shared-files", "E:/projects"]
```

**方式三：config.json 配置文件**

在程序同目录下创建 `config.json` 文件，可集中管理所有配置项：

```json
{
  "port": 8080,
  "password": "mypass",
  "roots": ["D:/shared"],
  "read_only": false,
  "prevent_sleep": true,
  "lang": "zh",
  "ssl_cert": "/path/to/cert.pem",
  "ssl_key": "/path/to/key.pem"
}
```

> 优先级：`config.json` < 命令行参数。即命令行参数会覆盖 `config.json` 中的同名配置。

配置后的效果：
- 首页只显示白名单中的目录（替代原来的磁盘列表）
- 所有 API 自动受限：浏览、搜索、下载、上传、复制、移动等操作均只能在白名单范围内进行
- 白名单外的路径一律拒绝访问（返回 404）
- 通过符号链接（symlink）绕过的尝试也会被拦截（使用 `os.path.realpath` 解析真实路径）

启动后终端会显示：
```
  Access:   2 allowed dir(s):
            - D:/shared-files
            - E:/projects
```

不指定 `--roots` 且 `ALLOWED_ROOTS = []` 则不限制（默认行为）。

## 只读模式

启用后，所有用户只能浏览、预览和下载文件，无法进行任何修改操作。适合教师分享课件、公司共享文档等场景。

**禁用的操作：** 上传、新建文件/文件夹、编辑、删除、重命名、复制、移动、解压

**保留的操作：** 浏览目录、文件预览、单文件下载、批量下载、文件夹下载、搜索、收藏、剪贴板读取、分享链接

**启用方式：**

方式一：启动时交互选择
```
  权限模式:
    1. 完全权限（默认）
    2. 只读模式
  请选择 [1]: 2
```

方式二：命令行参数
```bash
python file_browser.py --read-only
```

方式三：修改配置文件
```python
READ_ONLY = True
```

**典型教师场景：**
```bash
# 只分享课件目录，只读模式，无需密码（教室内网）
python file_browser.py --roots D:/课件 --read-only --no-password
```

启动后终端会显示 `Mode: READ-ONLY`，页面上写操作按钮自动隐藏。

## 多用户多密码

当需要不同用户有不同权限时（如教师管理 + 学生只读），可以配置多用户模式。

在 `file_browser.py` 顶部配置：

```python
USERS = {
    "teacher": {"password": "teacher123", "role": "admin"},
    "student": {"password": "stu456",     "role": "readonly"},
}
```

**角色说明：**

| 角色 | 权限 |
|------|------|
| `admin` | 完全权限（浏览、预览、下载、上传、编辑、删除、复制、移动等） |
| `readonly` | 只读权限（浏览、预览、下载、搜索，禁止任何修改操作） |

**行为：**
- 配置 `USERS` 后自动启用多用户模式，`PASSWORD` 配置项被忽略
- 登录页面输入对应用户的密码即可，无需输入用户名（系统自动匹配）
- 只读用户登录后，页面上的写操作按钮自动隐藏
- 终端 banner 正常显示，不会暴露用户密码

**典型教师场景：**
```python
# 教师用 teacher123 登录可管理文件
# 学生用 stu456 登录只能看和下载
USERS = {
    "teacher": {"password": "teacher123", "role": "admin"},
    "student": {"password": "stu456",     "role": "readonly"},
}
ALLOWED_ROOTS = ["D:/课件"]  # 只暴露课件目录
```

> 注意：多用户模式和单密码模式（`PASSWORD`）互斥。配置了 `USERS` 后 `PASSWORD` 不生效。

## 密码保护与登录

启动时自动生成 32 位高强度随机密码，在终端中显示。用户在浏览器输入密码后通过 cookie 认证，关闭浏览器后需重新登录。

三种密码模式（修改 `file_browser.py` 顶部的 `PASSWORD` 变量）：

| 设置值 | 行为 |
|--------|------|
| `PASSWORD = None` | **默认**。每次启动随机生成 |
| `PASSWORD = "my123"` | 固定密码 |
| `PASSWORD = ""` | 禁用密码保护 |

## 文件名搜索

1. 在顶部搜索框输入关键词，500ms 后自动搜索
2. 不区分大小写，支持部分匹配
3. 范围：当前目录递归 6 层，最多 100 条
4. 点击 ✕ 清除搜索

**技巧**：先进入目标目录再搜索，范围更小、速度更快。

---

## 文件内容搜索

1. 输入关键词后，搜索框下方出现 **"文件名搜索"** / **"内容搜索"** 切换按钮
2. 点击 **"内容搜索"** 搜索文件内部文字
3. 结果显示：文件名 + 匹配行号 + 行内容

限制：仅搜索文本/代码/Markdown 文件，单文件 512KB 以内，最多扫描 500 个文件。

## 正则搜索

搜索框右侧有一个 **`.*`** 按钮，点击启用正则表达式模式（高亮表示已启用）。

支持标准正则语法，例如：
- `\.py$` — 匹配以 .py 结尾的文件名
- `TODO|FIXME` — 匹配 TODO 或 FIXME
- `\d{4}-\d{2}-\d{2}` — 匹配日期格式

文件名搜索和内容搜索均支持正则模式。

## 视频字幕

播放视频时，程序会自动检测同目录下是否有与视频同名的字幕文件：

- 支持格式：`.vtt`、`.srt`、`.ass`
- 例如：播放 `movie.mp4` 时自动查找 `movie.vtt`、`movie.srt`、`movie.ass`
- 找到即自动加载，第一个字幕默认启用
- 如果有多种格式的字幕同时存在，播放器中可以切换

> `.vtt` 格式兼容性最好，建议优先使用。

## ZIP 预览与解压

点击 `.zip` 文件打开预览弹窗，可以：
- 查看 ZIP 内的完整文件列表和大小

> 其他压缩格式（.rar, .7z 等）不支持预览，需下载后本地解压。

## 上传文件与拖拽上传

**方式一：按钮上传（文件）**
1. 进入目标目录 → 点击工具栏 **"⬆ 上传"**
2. 点击 **"选择文件"** → 选择文件（支持多选）→ 开始上传

**方式二：按钮上传（文件夹）**
1. 进入目标目录 → 点击工具栏 **"⬆ 上传"**
2. 点击 **"选择文件夹"** → 选择一个文件夹 → 文件夹内所有文件连同目录结构一起上传

**方式三：拖拽上传**
直接将文件或**文件夹**从电脑桌面或文件管理器**拖到浏览器页面**，松开即上传。拖入文件夹时自动递归读取其中所有文件并保留目录结构。

- 上传过程中显示**实时进度条**（百分比 + 已上传/总大小），拖拽上传在页面底部显示浮动进度条
- 无文件大小限制
- 普通文件上传同名时，根据上传对话框中选择的处理方式自动处理（默认重命名加数字后缀，也可选择覆盖或跳过）

**大文件分片断点续传：**

当文件大小 ≥5MB 时，系统自动启用分片上传模式（每片 5MB）：

- **自动分片**：前端自动将大文件切分为 5MB 的分片，逐片上传
- **断点续传**：网络中断后重新上传，已成功的分片不会重复上传，从断点处继续
- **暂停/继续**：上传过程中可随时暂停，稍后继续上传
- **取消上传**：取消后自动清理服务端已上传的临时分片文件
- **进度显示**：实时显示总进度和当前分片进度
- **自动清理**：服务端临时分片文件 24 小时后自动清理，无需手动管理

> 拖拽上传和按钮上传均已升级为逐文件顺序上传模式，单个文件上传失败不影响其余文件继续上传。

## 文件复制

1. 点击文件/文件夹右侧的 **📋** 按钮
2. 弹出**可视化目录选择器**，显示当前目录的子文件夹
3. 点击文件夹可进入，点击 `⬆ ..` 返回上级，点击顶部输入框可手动输入路径
4. 在目标目录点击 **"确认选择当前目录"**

目标存在同名文件/文件夹时，弹出冲突对话框，可选择覆盖、重命名保留两者或跳过。

**实时进度条：** 复制、移动、删除（递归）、ZIP 解压操作均支持实时进度条显示：
- **大文件复制/移动**：显示字节级进度（如 45% 450MB/1GB），实时更新传输速度
- **递归删除**：显示已删除文件计数进度（如 150/300）
- **ZIP 解压**：显示已解压文件计数进度（如 75/200）
- **批量操作**：显示当前文件序号（如 [3/10]），每个文件各自显示独立进度

> 进度条通过 API 的 `stream=true` 参数启用 NDJSON 流式响应实现，详见 [API 文档](API.md#流式进度响应)。

## 文件移动

## 删除文件

点击 **🗑** 按钮 → 确认弹窗 → 删除。

- **文件**：直接删除
- **空文件夹**：直接删除
- **非空文件夹**：弹出二次确认（红色警告），确认后递归删除全部内容

> 删除不经过回收站，永久删除，请谨慎操作。系统关键目录（如 `C:\Windows`、`/usr`）受保护，无法删除。

## 临时分享链接

1. 预览任意文件时，点击弹窗顶部的 **"🔗 分享"** 按钮
2. 在下拉菜单中选择链接有效期（5 分钟 / 30 分钟 / 1 小时 / 6 小时 / 12 小时 / 24 小时）
3. 点击"生成分享链接"
4. 点击"复制链接"发送给他人
5. 对方打开链接即可直接下载，**无需登录**

> 链接过期后自动失效，返回 410 Gone。

## 共享剪贴板

适用场景：手机和电脑间传递网址、密码、代码片段等文本。

> 多用户模式下，剪贴板数据按用户隔离，每个用户只能读写自己的剪贴板内容。

## 常用目录收藏

- **添加**：进入目录后点击工具栏 **"+⭐"**
- **查看**：点击 **"⭐ 收藏"** → 点击目录名跳转
- **删除**：收藏列表中点击 ✕

收藏数据持久保存在 `bookmarks.json` 中，重启不丢失。

## 阻止系统睡眠

服务运行期间，程序默认会阻止电脑进入睡眠/休眠状态，确保手机等设备能持续访问。

**跨平台实现：**

| 平台 | 技术方案 | 说明 |
|------|---------|------|
| Windows | `SetThreadExecutionState` API | 通知系统有程序需要持续运行 |
| macOS | `caffeinate -i -w <pid>` | 系统自带命令，阻止空闲睡眠 |
| Linux | `systemd-inhibit --what=idle` | systemd 抑制锁机制 |

**行为：**
- 启动后终端显示 `Sleep: blocked` 表示生效
- 程序退出时（Ctrl+C 或关闭终端）自动恢复系统正常睡眠行为
- 不影响手动合盖睡眠或手动点击"睡眠"按钮

**关闭此功能：**
- 命令行：`python file_browser.py --no-sleep`
- 或修改文件：`PREVENT_SLEEP = False`

## HTTPS 配置

通过提供 SSL 证书和私钥文件即可启用 HTTPS 加密访问。

**方式一：命令行参数**

```bash
python file_browser.py --ssl-cert /path/to/cert.pem --ssl-key /path/to/key.pem
```

**方式二：config.json 配置文件**

```json
{
  "ssl_cert": "/path/to/cert.pem",
  "ssl_key": "/path/to/key.pem"
}
```

启用后，服务将通过 `https://` 提供访问，终端会显示 HTTPS 地址。

> 自签名证书会导致浏览器显示安全警告，可通过 Let's Encrypt 等服务获取免费可信证书。

## 公网访问（外网穿透）

默认情况下，服务只能在局域网内访问。如果你需要从外网（如在外面用手机访问宿舍电脑）访问，可以使用以下内网穿透工具。

> **安全提醒**：公网访问意味着任何人都可能尝试连接你的服务。请务必：
> - 保持密码保护开启（不要设置 `PASSWORD = ""`）
> - 配置 `ALLOWED_ROOTS` 白名单限制可访问目录
> - 用完后立即关闭穿透服务

### 方案一：Cloudflare Tunnel（推荐）

免费、自带 HTTPS、无需注册、速度快。

**第 1 步：安装 cloudflared**

- Windows: 下载 https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/ 并放到 PATH 中
- macOS: `brew install cloudflared`
- Linux: `sudo apt install cloudflared` 或从官网下载

**第 2 步：启动 File Browser**

```bash
python file_browser.py
```

**第 3 步：另开一个终端，创建隧道**

```bash
cloudflared tunnel --url http://localhost:25600
```

终端会输出一个公网地址，类似：

```
+----------------------------+
|  https://xxx-yyy.trycloudflare.com  |
+----------------------------+
```

将这个地址发给自己，在任何地方的浏览器中打开即可访问，自带 HTTPS 加密。

**关闭**：在隧道终端按 `Ctrl+C` 即可停止公网访问，File Browser 本身不受影响。

### 方案二：ngrok

业界最流行的穿透工具，免费版有带宽限制。

**第 1 步：注册并安装**

1. 访问 https://ngrok.com/ 注册免费账号
2. 下载 ngrok 并按官网指引配置 authtoken

**第 2 步：启动隧道**

```bash
ngrok http 25600
```

终端显示公网地址（如 `https://xxxx.ngrok-free.app`），复制使用即可。

### 方案三：SSH 隧道（零安装）

利用系统自带的 SSH 连接免费中转服务，无需安装任何额外工具。

```bash
ssh -R 80:localhost:25600 nokey@localhost.run
```

终端会输出一个 `https://xxxx.localhost.run` 地址。

> 稳定性一般，适合临时使用。Windows 10 及以上自带 SSH 客户端。

### 方案对比

| 方案 | 安装 | 注册 | HTTPS | 免费 | 稳定性 | 速度 |
|------|------|------|-------|------|--------|------|
| **Cloudflare Tunnel** | 需安装 cloudflared | 不需要 | 自带 | 完全免费 | 高 | 快 |
| **ngrok** | 需安装 ngrok | 需注册 | 自带 | 有限制 | 高 | 快 |
| **SSH 隧道** | 不需要 | 不需要 | 自带 | 完全免费 | 一般 | 一般 |

2026-03-05 14:34:00 | 192.168.1.50 | SHARE_CREATE | C:\Users\doc.pdf

记录的操作类型：LOGIN, BROWSE, PREVIEW, RAW, SEARCH, CONTENT_SEARCH, DOWNLOAD, DOWNLOAD_FOLDER, BATCH_DOWNLOAD, UPLOAD, MKDIR, MKFILE, DELETE, DELETE_RECURSIVE, RENAME, EDIT, COPY, MOVE, EXTRACT, SHARE_CREATE, SHARE_DOWNLOAD, CLIPBOARD, BOOKMARK_ADD, BOOKMARK_DEL, FOLDER_SIZE, ZIP_LIST。
